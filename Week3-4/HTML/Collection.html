HashMap
  • Time Complexity: O(1)
  • Unordered
  • Can have 1 null key, no null values
  • Implemented as a HashTable
TreeMap
  • Time Complexity: O(logN)
  • Ordered by key (using Comparable and Comparator)
  • No null can, but can have null values
  • Implemented as a Red Black Tree
    • An RBT is a Binary Search Tree modified to keep a balanced shape
    • Structuring rules increases time complexity
HashSet
  • Time Complexity: O(1)
  • Unordered
  • Can have 1 null value
  • Implemented using HashMap
TreeSet
  • Time Complexity: O(logN)
  • Sorted in Ascending order (using Comparable and Comparator)
  • No null values
  • Implemented using TreeMap
LinkedHashSet and LinkedHashMap
  • Both maintain insertion order, but implementation requires more memory than their HashSet and HashMap counterparts
  • LinkedHashSet is implemented using LinkedList and HashSet
  • LinkedHashMap is implemented using LinkedList and HashMap
